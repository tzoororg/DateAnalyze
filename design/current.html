Us ♥ — current design catalog

Every page and view of the app as it looks today (Sticker Book theme, Plum — the default). Screenshots are captured from the real app with demo data via node design/capture.mjs (needs the dev server running); re-run it after UI changes to refresh this catalog.

Tab 1 · Home

Home — tonight's pick, memories, recent dates

  • App chrome — the "Us ♥" header, ⋯ menu button and the four pill tabs (Home / Album / Ideas / Stats) are sticky; they stay visible while scrolling.
  • Tonight's pick — the top result from the suggestion engine, with a sticker tag ("favorite ♥" or "new!") and the reason it was picked. "Plan it →" jumps to the Ideas tab.
  • On this day — memory card shown when a past date matches today's month/day in a prior year; dismissible per session. Entries with a time-capsule note show it inside the card as a gold-bordered "💌 From you" panel, exactly one year after it was written.
  • Recent memories — the three latest entries as edge-to-edge photo cards (1 = full-bleed, 2 = 50/50, 3+ = big-left mosaic with "+N"; all the same height, emoji banner when photoless). Metadata sits ON the photo as stickers: date tape strip (top-left), category badge (top-right), cursive title + category·cost caption over a bottom scrim, hearts pill (bottom-right, my rating → partner's → legacy), and the vibe word as a small tape label when set. Tapping the card jumps to the Album tab with that date expanded — no inline Edit/Delete.
  • 🌙 Date night? card (between Tonight's pick and On this day) — "Start" begins a Date Night session (Roadmap #7). While active it's replaced everywhere by a pinned banner (moon glow, elapsed time · photo count, End pill) and the ＋ FAB becomes a camera that saves photos straight into the session. "End" clears the session and opens the log sheet pre-filled with date/duration/photos.
  • ＋ FAB (bottom-right) opens the full-screen log sheet from any tab; becomes a camera button during an active Date Night session.
Log sheet

"New page!" — the log form (＋ button)

  • Opens as a full-screen sheet over any tab; ✕ closes without saving.
  • Polaroid-first (v2): the photo well is the top of the form (tap → Camera / Gallery / Google Photos, first photo shown with a "+N" badge; a manage strip with ✕ appears below); the caption written on the polaroid's bottom edge IS the title.
  • Below: a date pill with the selected category's name beside it (cursive accent, first word only so "Movie / Show" → "Movie"); then the 11 category dots in a two-row grid directly under the date (all visible, no scroll); then cost tiers Free/$/$$/$$$ (no label — the $-signs speak for themselves; a representative ₪ lands in the numeric cost for analytics); and a free-text "one word for the vibe" with your most-used past words as suggestion chips (replaces the fixed mood list), pinned to the bottom of the column so it aligns with the again-o-meter's base.
  • Again-o-meter — vertical slider docked right of those fields (😵 never again → 😍 again ASAP). One drag sets enjoyment 1–5 AND wouldRepeat (4–5 yes, 3 maybe, 1–2 no); the thumb face and cursive verdict line update live. Effort and location were dropped from the form; notes hide behind "+ add a note" and an optional link (booking page, Pinterest…) hides behind "+ add a link" — set once on a wishlist idea, it carries through to the logged date. A butter "💌 note to next year" pill on the note row opens the time-capsule field (dashed gold box, "opens <date+1y>" caption); the note resurfaces in Home's On-this-day card a year later.
  • The same sheet is reused for editing an existing entry (title becomes "Edit date", adds Cancel).
  • Ending a Date Night session opens this same sheet with a mint "✨ From tonight — 1h 24m · 3 photos" chip at the top and the date/duration/photos already filled in — display-only, the rest of the form is unchanged.
Tab 2 · Album

Album — browse, search, filter

  • Controls card, collapsed to two compact rows: search + sort select on row one; icon-only List ⇄ Gallery ⇄ Wishlist toggle, ▶ Slideshow, and collapsible Category + Vibe filter chips (badges when set) on row two, with the filtered count pushed to the right.
  • Tapping a row expands it in place: a full-bleed photo hero (190px; single photo fills it, 2+ tile into the grid, "+N" badge, tap → lightbox), one wrapping chip row (moods + would-repeat; location joins the header meta line as "· 📍 …"), then per-partner star ratings — unrated shows a raspberry "Rate ★" pill that swaps into the 5-star input — notes, and a "Notes to each other" comment thread with input. Edit/Delete live in a ⋯ corner menu. Cost shows everywhere as one coarse Free/$/$$/$$$ badge; Date Night entries append their logged duration ("· 1h 24m").
  • The count under the filters ("19 dates") tracks the active filter.
  • Gallery view flattens every photo of the filtered entries into a grid with the entry title as caption.
  • Tapping a tile opens the lightbox with swipe/arrow navigation.
  • ▶ Slideshow plays a highlight reel (best-rated photo entries) with 5s auto-advance; tapping a slide opens that entry.
  • ☆ Wishlist segment (alongside List/Gallery) lists date ideas saved from the Ideas tab — title, category · cost badge · effort, and an optional link. Ideas are ordinary date docs with status:"idea", so they sync for free but are excluded from every stat, chart, suggestion and the normal list/gallery.
  • We did it! Log it → opens the log sheet pre-filled from the idea (same doc, keeps id + link) so saving converts it into a real logged date — no duplicate. Remove deletes it.
Tab 3 · Ideas

Ideas — the suggestion engine (UCB1 bandit)

  • Comfort ⇄ Adventure slider — the explore parameter: left repeats favorites, right pushes unseen catalog ideas. Results always mix both.
  • Filters: max budget (Free/$/$$/$$$ segmented pill, tap again to clear), max effort, category, vibe. "🎲 Surprise us" re-rolls with jitter; "📍 Find nearby" opens a maps search.
  • Each card: sticker ("Favorite"/"New"), reason line, category · $-tier · effort dots, and a compact one-line action row — "♡ Wishlist" (stores it as a status:"idea" date for the Album's Wishlist segment) and "Log →" (pre-fills the log sheet). Already-saved cards (matched by normalized title) show a "saved ♡" sticker and a disabled ✓ Saved button. The results list carries bottom padding so the FAB never covers the last card.
  • Cards of past favorites show their photos.
Tab 4 · Stats

Stats — insights & charts

  • Your Wrapped ✨: a shareable Spotify-Wrapped-style recap card at the top, with a This year / All time toggle. Shows dates count, avg♥, favorite category, most-repeated activity, "our usual" cost tier (mode + share, no ₪ sums), best month (full month name, e.g. "June"), and top vibe words. "Share this card" rasterizes it to a PNG via canvas and opens the OS share sheet (falls back to a download).
  • Stat grid: dates logged, average enjoyment (♥), typical date cost tier (mode of tiers — no summed ₪ anywhere), categories tried.
  • Charts are hand-rolled inline SVG themed by CSS variables: enjoyment by category (bars), your vibes (one merged list: emoji · avg★ + top-category icon · inline frequency bar · count), trend over time (enjoyment line vs. date count). The enjoyment-vs-cost scatter was removed in the cost-tier purge (Jul 2026).
  • Lists: best value for money (tier pill + hearts per row) and most repeat-worthy (repeats grouped by normalized title).
  • Adventure balance donut: how many of the 11 categories you've tried, and how exploratory recent dates were.
⋯ menu

Menu sheet — themes, data, sync

  • Theme switch: ☀️ Pink / 🍇 Plum (default) / 🌙 Navy.
  • 🎞️ Slideshow when idle: Off / 30 s / 1 min (default) / 2 min / 5 min — how long with no interaction before the highlights slideshow auto-starts. Per-device (localStorage), not synced.
  • Data: export JSON backup, import/restore, 📷 import from photos (EXIF-dated triage), sample data, erase everything.
  • Sync (rows appear per state): Google sign-in → create shared space / join with code → copy code, 🔔 push on partner's new dates, photo backfill, sign out. Local-only is the default; sync is opt-in.
  • 💬 Send feedback posts to the feedback Worker (becomes a GitHub issue).
  • Footer shows the service-worker build version.
